对象约束的声明与校验

声明对象约束

约束在 Jakarta Bean Validation 中是通过注解来表达的，对象约束分别4个类型：

  • 字段约束
  • 属性约束（属性：私有成员变量及其getter/setter）
  • 容器元素约束
  • 类约束

下面将分别介绍每种约束的声明方法

字段约束

当使用字段级别约束时，验证引擎会直接访问实例对象进行验证，而不会执行get方法。

约束可以被应用于任何访问级别的字段上（例如：public, private），但是不能用于静态字段。

属性约束

属性指的是私有成员变量及其getter/setter

当使用属性级别约束时，验证引擎会通过属性方法来访问被验证的值

容器元素约束

容器元素约束指的是直接在泛型的类型参数上设置的约束

Hibernate 支持以下标准Java容器的容器元素约束 ：

  • Iterable 接口实现
  • Map 接口实现
  • Optional

自定义的容器类型可以通过提供 ValueExtractor 接口实现来支持（TODO）

类级约束

约束继承

类实现接口 或者 继承另一个类时，字段和方法中的注解也会被继承下来，因此接口和父类中的约束会被继承到子类中

级联校验

Jakarta Bean Validation API 不仅仅支持单个对象实例的校验，还能完成整个对象图的校验。

要实现整个对象图的校验，只需要在另一个对象的字段引用上加上@Valid 注解即可。

校验对象约束

获取校验器

通过Validation 静态工厂方法可以轻松地创建一个校验器

校验器API

validate

最常用的校验API，能够对传入object对象的所有约束进行校验，并返回校验结果。

validateProperty

对object的propertyName属性进行校验，并返回校验结果

validateValue

使用beanType的propertyName属性的约束规则对值进行校验，并返回校验结果

方法约束的声明与校验

声明方法约束

参数约束

在方法参数上添加注解，可以声明对方法参数的约束

跨参数约束

如果方法的参数之间存在某些的关系，需要同时对多个参数的组合进行校验，那么就需要通过自定义约束来实现。（TODO）

返回值约束

在方法上添加注解，可以声明对方法返回值的约束

级联校验

跟对象的级联校验类似

  • 使用 @Valid 标识方法的参数就可以对参数进行级联校验

  • 使用 @Valid 标识方法本身就可以对返回值进行级联校验

如果要对容器中的每个元素进行校验，只需在容器的泛型参数上标识 @Valid

校验方法约束

获取校验器

校验API

validateParameters

使用object对象的methdo方法的参数约束，对parameterValues进行校验，返回校验结果

validateReturnValue

使用object对象的methdo方法的返回值约束，对returnValue进行校验，返回校验结果

validateConstructorParameters

使用constructor构造方法的参数约束，对parameterValues进行校验，返回校验结果

validateConstructorReturnValue

使用constructor构造方法的返回值约束，对createdObject进行校验，返回校验结果

使用消息表达式进行插值

从 Hibernate Validator 5 (Bean Validation 1.1) 开始，支持在约束违反信息中使用 Jakarta Expression Language 。这使得错误信息能够基于条件逻辑定义，并且支持高级格式选项。校验引擎在EL上下文中提供以下对象：

  • 约束注解的属性值
  • 当前验证的值（属性，Bean，方法参数等），即 validatedValue 变量
  • 格式器对象，即 formatter 对象
