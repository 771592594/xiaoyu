Java-等待通知机制

发表于 2021-11-01 更新于 2021-11-07 分类于 技术分享

前言

在之前写的 《Java-锁粒度与死锁问题》 一文中，我们知道死锁有四个必要条件，只要破坏其一就不会出现。在破坏”占有并等待“条件时，我创建了一个资源分配器，只有当线程能够获取到要求全部资源时，才会将资源分配给它；如果线程无法获取到所有的资源，则会使用死循环来不断尝试获取。

如果线程申请资源的时间较短且并发冲突不大的情况下，这种循环等待的方式是不错的；但是在申请时间较长或并发冲突量大的情况下，会导致CPU长时间的空转，浪费系统的资源。

在这种场景下，最好的做法应该是使用等待-通知机制：当要求的条件无法满足时，线程先挂起自己，进入等待状态。当条件可以满足时，再通知线程执行任务。这样才能最大限度地利用CPU的资源。

Java-内存模型如何解决可见性问题

发表于 2021-10-24 更新于 2021-11-06 分类于 技术分享

前言

在《并发Bug的根源-可见性、原子性和有序性问题》文章中，我们知道导致可见性问题的根源是缓存机制，有序性问题的根源是编译优化。如果我们禁用缓存和编译优化，这些问题就可以解决，但程序的性能就堪忧了。

因此最好的办法就是 按需禁用缓存以及编译优化。那么，如何做到“按需禁用”呢？对于并发程序，何时禁用缓存以及编译优化只有程序员知道，那所谓“按需禁用”其实就是指按照程序员的要求来禁用。所以，为了解决可见性和有序性问题，只需要提供给程序员按需禁用缓存和编译优化的方法即可。

Java 内存模型规范了 JVM 如何提供按需禁用缓存和编译优化的方法。具体来说，这些方法包括 volatile、synchronized 和 final 三个关键字，以及 Happens-Before 规则，这也正是本期的重点内容。
