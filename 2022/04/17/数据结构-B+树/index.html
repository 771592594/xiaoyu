数据结构-B+树

发表于 2022-04-17 更新于 2022-04-18 分类于 技术分享
本文字数： 1.9k 阅读时长 ≈ 5 分钟

前言

B+树是一种平衡的多叉树，通常用于实现”数据库”和”文件系统”的索引。我们所熟知的 innodb 存储引擎的索引就是使用B+树来实现的。为了更好地理解它的索引机制，我花了几天地时间学习了 B+ 树，并且成功地实现了一个基于内存的B+树，在此做一个简单的分享和总结。

如何学习B+树

前置准备

学习B+树前，需要掌握以下内容：

  • 链表的性质和基本操作
  • 二叉搜索树的性质和基本操作

学习方法

B+树操作相对来说比较复杂，学习时可以按照下面的步骤循序渐进：

  1. 了解数据结构和性质
  2. 学习查询/插入/删除操作
  3. 给定一组数据，在草稿纸上一步一步地画出B+树
  4. 在 Github 上找一个能跑的B+树代码，通过调试来学习具体实现细节。这里放一个我学习时参考的代码：https://github.com/jiaguofang/b-plus-tree.git（这个代码不是100%正确，但是给我了不少启发）
  5. 尝试用自己擅长的语言去实现基于内存的B+树
  6. 挑战项：实现基于磁盘的B+树

数据结构

B+树中有个概念叫”阶”，它表示节点能够记录的关键字的最大个数。例如：一颗3阶的B+树每个节点最多能够记录3个关键字。

下图是一颗3阶的B+树，从图中我们可以看出叶子节点和非叶子节点（后文称为 索引节点）的结构是有些不同的，所以我们需要先了解两种节点的共性和特性。

节点的共性

叶子节点和索引节点都满足一下性质：

  1. 对于阶为 M 的B+树，节点至多保存 M 个 有序排列 的关键字
  2. 对于阶为 M 的B+树，非根节点的叶子节点至少有 ceil(M) 个关键字；例如，3阶的B+树中非根节点至少有ceil(3) = 2 个节点

叶子节点的特性

B+树的叶子节点满足以下性质：

  1. 叶子节点维护关键字与数据的映射关系
  2. 叶子节点都在B+树的同一层，它的 next 指针指向比它大的下一个叶子节点，以此形成一条有序的链表（也可以增加一个 pre 指针实现双向链表）

索引节点的特性

B+树的索引节点满足以下特性：

  1. 索引节点维护关键字与孩子节点的映射关系

  2. 节点中任意的关键字 Ki 都必须是孩子节点 Ci 中最大/最小的关键字

  3. 索引节点有 n 个关键字，就必有 n 个孩子

  4. 若索引节点为根节点，则至少有 2 个关键字

等值查询

虽然B+树是多叉树，但它的等值查询和二叉搜索树的原理相似。

二叉搜索树是从根节点开始，将查询的关键字与当前节点的关键字进行比较，如果比当前节点小则访问左孩子；如果比当前节点大，则访问右孩子。重复此过程，直到找到关键字所在的节点或者到达空指针为止。

B+树的数据只保存在底层的叶子节点中，所以查询时需要通过索引节点不断向下访问，直到到达叶子节点。然后在叶子节点中查询关键字对应的数据。

插入

在B+树中插入关键字时，需要注意以下几点：

  • 插入的操作全部都在叶子结点上进行，且不能破坏关键字自小而大的顺序
  • 插入操作可能会出现结点中关键字个数超过阶数的情况，此时需要将对节点进行”分裂”操作
  • 父节点中的关键字必须是每个子节点的关键字中的最大值，如果叶子节点插入的关键字比原有的关键字都大，那么需要修改父节点中对应的关键字。

根据以上注意事项，B+树的插入操作可以分为以下四步：

  1. 找到合适的叶子节点
  2. 在叶子节点的合适位置插入键值
  3. 若插入后节点的关键字个数大于B+数的阶数，则对插入的节点进行拆分，修复B+树
  4. 修改父节点中对应的关键字

下面我通过几个案例来演示B+树插入关键字时会出现的几种情况：

一. 被插入的节点关键字个数小于阶数

案例1：根节点是叶子节点

第一次插入<1, A>，由于节点中没有关键字，所以直接插入数据

第二次插入<3, C>，因为关键字3比1大，所以插入到1后面

第三次插入<2, B>，因为关键字2比1大且比3小，所以插入到1和3的中间

案例2：向有父节点的叶子节点插入，插入的关键字比原有关键字都大

在B+树中插入<5, E>，根据索引节点找到 [3, 4] 节点并插入。插入后节点中最大的关键字变成了5，需要将父节点中的4也改成5

二. 插入节点的关键字个数等于阶数

当节点关键字个数已经超过了阶数，需要通过分裂节点修复B+树的性质，根节点和非根节点的分裂步骤有少许的不同，下面通过两个案例讲解。

案例1：根节点的分裂

根节点分裂的步骤如下：

  1. 将节从中间开始截断，将后半部分拆分为新节点
  2. 创建一个索引节点
  3. 将原节点与新节点收为索引节点的孩子
  4. 将原节点与新节点的最大的关键字添加到索引节点中

案例2：非根节点的分裂

非根节点与父节点唯一的不同就是，它的父节点就是它的索引节点，因此可以省略创建索引节点这一步。

非根节点的分裂步骤：

  1. 将节从中间开始截断，将后半部分拆分为新的节点
  2. 将分裂出的两个节点收为索引节点的孩子
  3. 将原节点与新节点的最大的关键字添加到索引节点
