Java-基本类型与包装类

发表于 2021-06-15 更新于 2022-07-13 分类于 技术分享
本文字数： 2k 阅读时长 ≈ 5 分钟

前言

Java 中数据类型可以分为 基本数据类型 和 引用数据类型 (创建的实例称为对象)，每个基本数据类型都有各自的引用类型的包装类，具体如下表所示：

基本类型 包装类
byte Byte
short Short
int Integer
long Long
float Float
double Double
char Character
boolean Boolean

作为一门面向对象的语言，Java 为什么有基本数据类型这种非对象的类型？

原因很简单：Java中基本数据类型保存数据本身；而包装类（也就是对象）是由对象头、实例数据、对齐填充三大部分组成。因此使用基本数据类型所需的内存空间较小，在性能上会比使用包装类要好。

包装类与基本类型的区别

一、基本数据类型只包含了数据；包装类是对基本类型的封装，它包含了数据和相关的方法。

二、基本类型不允许为 null，包装类可以赋值为null。在实际开发中，数据库表中的字段大多用 null 表示没有数据，所以数据表对应的实体类往往使用支持 null 的包装类来增强表现力。

三、进行数值比较时，基本类型使用 ==，包装类使用 equals() 方法。

数据缓存

为了提高程序的运行效率，Java 为非浮点型的包装类都提供了缓存。在对基本数据类型进行装箱时，如果使用的数值在缓存中存在，则不创建新的对象，而是直接使用缓存中的对象。包装类缓存的数值范围如下所示：

类型 缓存范围
Boolean (true, false)
Byte [-128, 127]
Short [-127, 128]
Integer [-128, 127]
Long [-128, 127]
Character [0, 127]

==与equals方法

== 对基本类型和引用类型类的相等判定是不同的：对于基本数据类型，== 会进行值比较；而对于引用类型，== 会比较对象的内存地址。包装类属于引用类型，在不使用数据缓存的情况下，两个对象的内存地址必然不同，因此最好使equals 方法进行相等判断。

equals 是 万类之父 Object 中的方法。在没有重写的情况下与 == 作用相同。但是为了正确地比较包装类的值，所有的包装类中都重写了equals 方法，对于同类型的对象会通过值比较来判断相等，非同类型的对象则判断为不相同。
