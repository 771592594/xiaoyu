算法-浅谈时间复杂度与空间复杂度

发表于 2021-06-20 更新于 2022-02-13 分类于 技术分享
本文字数： 2k 阅读时长 ≈ 2 分钟

前言

在大学的教课书告诉我们，一个算法的优劣是通过它的复杂度来评判的。算法复杂度分为时间复杂度和空间复杂度：时间复杂度是指执行算法所需要的计算工作量；而空间复杂度是指执行这个算法所需要的内存空间。

复杂度一般使用大O表示法来表述，括号中一般使用 n 来表示数据规模。如果输入的数据为数组、字符串或集合，那么 n 代表它们的元素个数或长度，如果输入的数据为一个数字，那么 n 代表的数字大小。

时间复杂度

分析时间复杂度只要分析在不同的数据规模下，算法的执行次数。

常见时间复杂度表示从小到大排序如下：

  • O(1)
  • O(log n)
  • O(n)
  • O(n2)
  • O(n3)
  • O(2n)
  • O(n!)

由此可以看出，每一层的节点展开后，要计算的节点树是原来的两倍。如果递归树有n层，那么要计算的节点个数最多为 2n-1（满二叉树的节点个数），因此斐波那契数列递归算法的时间复杂度是2的指数级的，也就是O(2n)。

最后提一下，计算递归算法时间复杂度的利器：主定理 Master Theorem。任何分治和递归的算法都可以通过主定理来计算。

空间复杂度

空间复杂度就比较简单了，可以直接根据算法中使用的额外变量就可以判断；如果是递归算法，则是递归栈的最大深度。如果在一个递归算法中使用了额外的对象，那么空间复杂度就是栈的最大深度和额外对象空间之间的较大者。
