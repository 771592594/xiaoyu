Java-缓冲流的实现与性能分析

发表于 2021-08-17 更新于 2021-11-26 分类于 技术分享
本文字数： 5.2k 阅读时长 ≈ 9 分钟

前言

在使用IO流时，我经常会习惯性地包装一层缓存流，因为上网的案例都是这么写的。例如，向文件写入一串文本时，我会创建一个 FileOutputStream，然后再用 BufferedOutputStream 进行包装。

对于缓存流，很多人只知道它内部提供了一个缓冲区，能够提高程序的运行效率。那么缓冲流的原理究竟是什么呢？使用缓存流真的比直接使用节点流效率更高吗？下面让我通过分析源码和实例测试来解答这两个问题。

装饰器模式

流按照角色可以分为：节点流 和 处理流。

  • 节点流是直接向节点读写数据的流，例如：读文件的 FileInputStream、读内存的 ByteArrayInputStream

  • 处理流是对节点流进行包装以提供额外功能的流，例如：基于缓冲区进行读取的 BufferedInputStream

Java 的IO流使用了设计模式中的 装饰器模式，下面是IO流的类关系图：

FilterInputStream 及其子类是包装类，它们的内部封装了节点流，通过重写和扩展方法，对节点流的功能进行增强。

本文要讲的缓冲输入流 BufferedInputStream 和 缓冲输出流 BufferedOutputStream 就是其中两个包装类，下面让我们通过源码和案例来分析它们实现原理和性能。

BufferedInputStream

实现原理

BufferedInputStream 内部维护一个字节数组作为缓存区，并且记录缓冲区的大小和下次读取的位置。创建时包装节点流，并初始化缓冲区大小。

它的 read() 方法会对缓冲区进行读取操作，如果缓冲区中没有可以读取字节，则先填充缓冲区再读取。

填充缓冲区是通过 fill() 方法实现的，它底层是调用节点流的本地方法 readBytes() 将磁盘/网络的数据写入缓冲区中

综上所述，缓冲输入流能够将一大块的数据填充到缓冲区（也就是内存）中，后续的读取直接从缓冲区返回数据，避免频繁读取磁盘/网络，从而提高程序的运行效率（因为内存的读取速度远快于磁盘/网络）。

性能测试

测试说明：分别使用文件流和包装后的缓冲流，从硬盘中读取1GB大小的文件，每次读取相同的字节数。缓冲流使用默认的8192大小的缓存区。

下表是两个流在每次读取不同字节数时所花的时间占比，可以看出在缓冲区大小大于读取的字节数时，缓冲流的效率远高于字节流；但是当读取的字节数接近或超过缓冲区大小时，效率上几乎没有差别。

读取字节 文件流 缓冲流
1024 78% 22%
2048 67% 33%
4096 57% 43%
8192 50% 50%
16384 50% 50%
32768 50% 50%

出现这种现象的原因是：随着读取字节数变大时，缓冲区的利用率下降，读取节点的次数变多（在长度为8192的缓冲区中，每次读取1024个字节时，缓冲区够读取8次；每次读取2048个字节时，缓冲区只够读4次；如果读取的字节数超过缓冲区大小时，那么缓冲区就失去了作用）

BufferedOutputStream

实现原理

BufferedOutputStream 内部也同样维护一个缓冲区，并记录缓冲区的有效字节数

它的 write() 方法会直接缓冲区进行写操作，如果缓冲区已满，则先输出缓冲区中的内容再写入。

flush 方法底层调用节点流的本地方法，将数据写入到磁盘/网络中

综上所述，缓冲输出流会一般会对缓冲区进行写操作，缓冲区写满了再通过节点流输出，这样可以减少磁盘/网络，从而提高程序的运行效率。

性能测试

测试说明：分别使用文件流和包装后的缓冲流，将1GB大小内容写入到文件中，每次写入相同的字节数。缓冲流使用默认的8192大小的缓存区。

下表是两个流在每次写入不同字节数时所花的时间占比，可以看出在缓冲区大小大于读取的字节数时，缓冲流的效率远高于字节流；但是当读取的字节数接近或超过缓冲区大小时，效率上几乎没有差别。

写入字节 文件流 缓冲流
1024 80% 20%
2048 68% 32%
4096 62% 38%
8192 50% 50%
16384 50% 50%
32768 50% 50%

出现这种现象的原因是：随着写入字节数变大时，缓冲区的利用率下降，写入节点的次数变多（在长度为8192的缓冲区中，每次写入1024个字节时，缓冲区够写入8次；每次读取2048个字节时，缓冲区只写入4次；如果读取的字节数超过缓冲区大小时，那么缓冲区就失去了作用）

总结

我们都听过木桶效应，木桶的最大容量取决于它最短的那块板，程序运行的速度也同样取决于运行过程中最慢的操作。缓冲流的核心思想就是利用内存读写速度快于 IO 的特性，以空间换时间来提高程序的运行速度。这种思想在计算机中运用非常广泛，比如：CPU寄存器的读写远快于内存，但程序运行又需要从内存中获取数据，为了平衡两者之间的差距，现代计算机在二者之间加入了一层读写速度接近CPUI的高速缓存，CPU可以直接从缓存缓存中获取数据，从而提高运行的效率。
