Mybatis-缓存机制

发表于 2021-12-22 更新于 2022-01-08 分类于 技术分享
本文字数： 3.6k 阅读时长 ≈ 7 分钟

前言

在高并发的场景下，数据库的查询往往会成为一个性能的瓶颈。但实际上很多的查询是相同，在数据没有被修改的情况下返回的结果也是相同，如果我们对查询的结果做一个缓存，就可以减少数据的压力。

Mybatis 是当前主流的持久层框架，它提供了两级的缓存机制，会将查询的数据自动保存到缓存中，后续相同的查询会直接从缓存中读取数据。下面让我们了解一下其缓存机制的使用方式和实现原理。

Mybatis的缓存

在 mybatis 中，所有与缓存相关的类都存放在 cache 包下。而且定义了一个顶级接口 Cache ，它虽然有很多的子类，但真正实现缓存行为的类只有 impl 目录下的 PerpetualCache；而 decorators 目录下的类都是装饰器（装饰器对实现类进行包装，对原有的方法进行增强）。

PerpetualCache

作为默认的唯一缓存实现类，PerpetualCache 的源码是非常简单的：它的字段只有一个 id 和一个名为 cache 哈希表。浏览它的方法，我们基本可以确定：这个缓存是一个 HashMap 实现的本地缓存。

实现原理

因为一级缓存的作用域是 SqlSession 级别的，因此它最适合存放的地方就是 SqlSession 的内部，因此我们可以尝试在 DefaultSqlSession 中找找看。它一共有如下五个字段，最后三个字段肯定是不可能存放缓存的，Configuration 保存的是全局配置也不适合存放，因此缓存最有可能存在于SQL的执行器 Executor 中。

在 DefaultSqlSessionFactory 的openSessionFromConnection 方法中，我们可以看到每次打开 SqlSession 时，都会创建一个新的 Executor 给 DefaultSqlSession 使用。

在 Executor 的抽象类 BaseExecutor 中，我找到了缓存对象。通过构造方法可知，当 Executor 对象被构造时会创建一个缓存对象。
