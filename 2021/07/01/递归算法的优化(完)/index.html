递归算法的优化

发表于 2021-07-01 更新于 2021-10-10 分类于 技术分享

前言

递归是一个非常考验程序员水平的编程思想，好的递归可以像数学公式一样简介优雅地描述一段过程，而暴力的递归则容易出现栈溢出或执行效率低下的问题。下面我将从 空间 和 效率 两方面分析递归算法的优化策略。

空间优化

递归算法依赖于方法栈，而编程语言对方法栈的深度一般都是有限制的。如果方法的递归深度过大，就会出现栈溢出异常，产生的原因有以下两点：

  1. 代码没有正确地设置程序终止条件，导致程序无限递归
  2. 输入的数据过大，导致程序递归深度达到极限

效率优化

递归算法效率低下往往是由以下两个原因造成：

  1. 存在大量重复计算的内容
  2. 计算过多的无效数据

针对这两大原因，我在下文提供两个优化案例供大家参考。

递归树剪枝

我们先看一道算法题：LeetCode 22. 括号生成

题目描述：数字 n 代表生成括号的对数，请你设计一个函数，生成所有可能的并且有效的括号组合。

这题最简单的解题思路就是枚举出所有的括号组合并筛选保留有效的。

在编码时，我们将生成的括号字符串看成一个个的格子（生成n对括号，对应 2n 个格子），每个格子只能放一个 ( 或 )，因此我们总共要列出 22n 种组合。我们通过递归函数每次向一个格子中放入括号，然后进入下一个格子执行同样的操作。如果格子全部都放满了，则判断生成的括号组合是否符合要求，将符合的括号组合加入结果集，并结束递归。代码如下所示：

虽然这个解题思路非常简单，但是执行效率非常低。执行效率低下的原因在于低效的枚举和检验，在这个方法枚举了全部的组合，但有些组合显示是不符合要求的，根本没有必要进行枚举（比如：左右括号数量不一样的组合）。

要提高这个算法的效率应该从枚举的实现入手：如果方法每次枚举只生成符合条件的组合，那么枚举次数将大大减少，并且可以省去检验的过程。

一个符合条件的括号组合应该满足以下条件：

  1. 左括号数 = 右括号数 = 括号的对数
  2. 右括号生成时，左侧必须存在没有配对的左括号（不当单身狗）

所以，我们修改枚举的实现方式：

  1. 只有当左括号数小于要生成的括号对数时，才生成左括号
  2. 只有当右括号数小于左括号数时，才生成右括号

文章总结

最后对本文进行一个总结：

  1. 递归算法依赖于方法栈，每次执行递归都有一个方法入栈。如果递归过于深入，就会存在栈溢出的风险，此时迭代法来代替非递归法
  2. 递归是一种特殊的迭代，所有的递归都可以通过迭代实现，反之亦然
  3. 要编写高效的递归算法，应该抓住需求的重点，减少无用的递归
  4. 如果递归算法在执行中，对一些数据进行了重复的计算，应当引入缓存，用空间换时间，避免重复计算。
  5. 在语言支持的情况下，可以进行尾递归优化减少栈空间的使用，提高运行效率
